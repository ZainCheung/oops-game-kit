## 一、项目概述

### 1.1 项目定位

《疯狂神射手》是一款以**第一人称模拟狩猎**为核心的 3D 射击类微信小游戏，通过分阶段迭代开发，逐步验证核心玩法、养成系统、竞技系统、社交系统和商业化能力，最终实现完整产品上线。

### 1.2 开发策略

采用 **MVP（最小可行产品）+ 快速迭代** 策略：

- **先验证核心玩法**：确保基础体验有吸引力
- **逐步叠加系统**：每个版本增加核心系统，验证用户反馈
- **数据驱动决策**：通过用户数据验证每个版本的有效性
- **快速试错调整**：根据数据反馈快速调整开发方向

### 1.3 技术架构

| 技术栈      | 说明                            |
| -------- | ----------------------------- |
| **游戏引擎** | Cocos Creator 3.8.8（支持 3D 渲染） |
| **开发语言** | TypeScript / JavaScript       |
| **平台**   | 微信小游戏                         |
| **后端**   | Node.js / Go / Java 等主流技术栈（用户数据管理、实时匹配对战、排行榜系统、社交关系存储） |
| **服务器**  | 云服务器（按需扩容）                    |

| **v0.1** | 不扩张 | 核心团队，AI辅助效率高         | -           |
| **v0.2** | 微扩张 | 新增养成系统，前端工作量增加       | +1 前端       |
| **v0.3** | 微扩张 | 新增竞技系统，需要后端支持        | +1 后端       |
| **v0.4** | 微扩张 | 新增社交系统，需要后端支持        | +1 后端       |
| **v0.5** | 不扩张 | 商业化系统复杂度可控，AI辅助效率高   | -           |
| **v1.0** | 微扩张 | 正式上线，需要美术完善和性能优化     | +1 美术       |

**扩张原则**：
- **优先使用AI辅助**：新功能优先考虑AI辅助开发，减少人力扩张
- **前端美术灵活扩张**：前端和美术岗位根据实际情况，为提高效率可以灵活扩张
- **核心岗位稳定**：策划、PM保持稳定，避免频繁扩张
- **按需扩张**：仅在AI辅助无法满足需求时扩张团队
- **灵活调整**：根据实际开发进度和AI效率灵活调整团队规模
