# 《疯狂神射手》复刻项目开发计划

> 文档版本：v3.0（AI辅助开发版 - 微信小游戏单版本）
> 整理日期：2026-06-22
> 基于文档：《疯狂神射手_最新版本玩法需求文档.md》（v2.0）
> 项目类型：微信小游戏 / 3D 模拟狩猎 / 狙击竞技
> 目标平台：微信小游戏
> 建议引擎：Cocos Creator 3.x（TypeScript）

## 一、项目整体安排

| 阶段 | 周期 | 核心目标 | 主要交付 |
|------|------|----------|----------|
| 第一期 | 4-5周 | MVP核心验证 | 核心战斗、主线PVE、基础PVP、付费/广告闭环 |
| 第二期 | 4-5周 | 内容扩充与养成深化 | 狩猎挑战、高山/自由猎场、配件/宝石/卡牌/养殖场 |

| **总计** | **11-14周（约3-3.5个月）** | | |

## 三、技术架构

### 3.1 技术选型

| 层级 | 方案 |
|------|------|
| 游戏引擎 | Cocos Creator 3.8+（TypeScript） |
| 物理 | 自研弹道物理 + Cannon.js 轻量碰撞 |
| 网络 | WebSocket + Protobuf |
| 后端语言 | Node.js / NestJS |
| 数据库 | MySQL 8.0 + Redis 6 |

## 四、第一期（4-5周）：MVP核心验证

### 4.1 阶段目标

完成核心战斗、主线PVE闯关、基础PVP竞技、基础养成与付费/广告闭环。

### 4.2 第1周：预研 + 框架 + 核心战斗原型

#### 4.2.1 技术预研与框架搭建（3天）

**负责人**：客户端主程、后端主程

**工作内容**：
- 确定Cocos Creator 3.8 + TypeScript工程结构
- 搭建客户端基础框架：场景管理、UI管理、资源管理、网络层、配置表解析
- 搭建后端NestJS框架：网关、登录、数据库、Redis、日志
- 建立Git + Git LFS仓库
- 完成微信登录、支付、广告SDK接入预研

**交付物**：可运行的空工程、技术方案文档

#### 4.2.2 核心战斗原型（3天）

**负责人**：客户端主程、3D美术

**工作内容**：
- 实现3D瞄准：开镜、滑动、准星
- 实现射击：弹道、下坠、风速、命中判定
- 完成1种动物（如鹿）的模型、动画、AI、受击/死亡
- 完成计分系统：击杀分、精准分、总分

**交付物**：单场景射击Demo，含1把枪、1种动物、基础计分

#### 4.2.3 美术管线与数值模板（1天）

**负责人**：3D美术、2D/UI美术、策划

**工作内容**：
- 确定动物、场景、枪械、UI风格规范
- 建立数值模板Excel：动物血量、防御、分数、枪械伤害、经济曲线
- 完成主界面、战斗界面的UI原型

**交付物**：美术规范文档、数值模板、UI原型

### 4.3 第2周：主线PVE + 基础养成

#### 4.3.1 主线狩猎场（4天）

**负责人**：客户端主程、策划

**工作内容**：
- 实现关卡配置表：动物组合、血量、奖励、解锁条件
- 完成探索机制：随机奖励、三个标靶进入狩猎
- 完成30关主线狩猎场
- 完成通关判定、奖励发放、关卡进度存储

**交付物**：可连续闯关30关的主线PVE

#### 4.3.2 装备与经济系统（3天）

**负责人**：客户端主程、后端主程

**工作内容**：
- 实现枪械系统：升级、升阶、装备/卸下
- 实现子弹系统：红/橙/蓝子弹及其加成
- 实现金币、钻石、宝箱、抽卡
- 完成背包与商城基础功能

**交付物**：可强化枪械、购买宝箱、抽卡的养成闭环

### 4.4 第3-4周：PVP + 排行榜 + 付费/广告

#### 4.4.1 竞技场PVP（5天）

**负责人**：客户端主程、后端主程

**工作内容**：
- 实现1v1匹配：基于奖杯/战力的ELO匹配
- 实现三局制对战、总分胜负判定
- 实现奖杯增减、竞技场关卡解锁
- 完成排行榜：全服/好友实时排名

**交付物**：可匹配的1v1 PVP和排行榜

#### 4.4.2 付费与广告变现（3天）

**负责人**：客户端主程、后端主程

**工作内容**：
- 接入微信登录、支付、米大师
- 实现首充礼包（6/30/68元三档）
- 接入微信激励视频、插屏、Banner广告
- 实现广告奖励：体力补充、额外挑战次数、免费宝箱

**交付物**：可付费、可看广告获得奖励的完整闭环

#### 4.4.3 任务与活动系统（3天）

**负责人**：后端主程、策划

**工作内容**：
- 实现日常任务、周常任务、七日签到
- 实现基础活动系统（首充、签到）
- 实现邮件系统发放奖励

**交付物**：任务、签到、邮件系统

#### 4.4.4 美术与UI量产（并行进行）

**负责人**：3D美术、2D/UI美术

**工作内容**：
- 完成8-12种动物模型、3-4个场景、5-8把枪械
- 完成主界面、背包、商城、战斗UI
- 完成开火、命中、UI动效

**交付物**：MVP核心美术资源包

#### 4.4.5 MVP测试与优化（5天）

**负责人**：测试（兼职）、客户端主程

**工作内容**：
- 功能测试、性能测试、兼容性测试
- 优化首包大小（< 4MB）、运行帧率（30fps+）
- 修复阻塞性Bug

**交付物**：MVP可测试安装包、测试报告

## 五、第二期（4-5周）：内容扩充与养成深化

### 5.1 阶段目标

扩充PVE玩法内容，完善配件、宝石、卡牌、养殖场等养成系统。

**负责人**：客户端主程、策划

**工作内容**：
- 实现波次生存玩法：击败所有动物进入下一层
- 完成每日3次、购买次数、广告次数
- 实现排行榜：最高通关关卡记录
- 配置50关狩猎挑战

**交付物**：可玩的狩猎挑战及排行榜

#### 5.2.2 高山狩猎场（3天）

**负责人**：客户端主程、3D美术

**工作内容**：
- 完成高山场景与10关关卡
- 实现高山水晶产出
- 实现黑曜石钥匙兑换与配件宝箱开启

**交付物**：高山猎场与配件宝箱系统

#### 5.2.3 配件系统（4天）

**负责人**：客户端主程

**工作内容**：
- 实现5部位配件：弹夹、握把、枪口、枪托、倍镜
- 实现配件升级、合成、品质（含彩色）
- 实现特殊配件效果：S·幽灵静音弹夹、透视倍镜等

**交付物**：完整的配件系统

#### 5.2.4 自由狩猎场（2天）

**负责人**：客户端主程、策划

**工作内容**：
- 完成自由狩猎场景3阶
- 实现无拘束探索与特定资源产出

**交付物**：自由狩猎场

### 5.3 第7-8周：宝石 + 卡牌 + 养殖

#### 5.3.1 宝石系统（4天）

**负责人**：客户端主程、策划

**工作内容**：
- 实现5部位7品级宝石
- 实现镶嵌、合成、洗炼、锁定规则
- 实现宝石属性：伤害、积分、体型、类型、枪械加成等

**交付物**：完整宝石系统

#### 5.3.2 卡牌/图鉴系统（3天）

**负责人**：客户端主程、2D/UI美术

**工作内容**：
- 实现2-3个栖息地（如灌木林、雪原）的卡牌
- 实现卡牌收集、升级、分数加成
- 完成卡包抽取与图鉴展示

**交付物**：卡牌/图鉴系统

#### 5.3.3 养殖场系统（4天）

**负责人**：客户端主程

**工作内容**：
- 实现普通养殖场与猛兽养殖场
- 实现幼崽成长、加速药剂、成年出售
- 实现好友拜访、偷取动物、互赠体力
- 实现养殖场彩蛋（清理石头/便便）

**交付物**：养殖场系统

#### 5.3.4 运营活动扩展（2天）

**负责人**：后端主程、策划

**工作内容**：
- 实现每周福利礼包码系统
- 实现通行证基础框架
- 实现邀请有礼（最多5人）

**交付物**：礼包码、通行证、邀请活动

### 5.4 第9周：内容量产 + 数值调优

#### 5.4.1 内容量产（持续进行）

**负责人**：3D美术、2D/UI美术、策划

**工作内容**：
- 补充动物至20-30种、场景至5-7个、枪械至15-20把
- 扩展主线狩猎场至80-100关
- 扩展狩猎挑战、高山猎场关卡

**交付物**：内容量接近完整版的资源包

#### 5.4.2 数值与经济调优（3天）

**负责人**：策划、客户端主程

**工作内容**：
- 建立数值模拟工具，验证伤害、分数、经济曲线
- 调整关卡难度与玩家成长节奏匹配
- 调整PVP匹配公平性与胜率

**交付物**：数值平衡报告

#### 5.4.3 性能与兼容性优化（3天）

**负责人**：客户端主程、测试

**工作内容**：

- 主流机型兼容性测试

**交付物**：优化后的客户端包与测试报告

## 九、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 3D性能不达标 | 高 | 早期做性能Demo，持续监控，使用LOD、动态加载、贴图压缩 |
| 动物AI表现差 | 高 | 简化AI，优先行为树 + 动画状态机 |
| 美术资源不足 | 高 | 先完成核心动物与场景，逐步扩展 |
| 网络同步延迟 | 中 | PVP采用关键事件同步 + 服务器仲裁；PVE本地计算 |
| 数值平衡崩坏 | 高 | 建立数值模拟工具，小范围测试 |
| 微信审核不通过 | 中 | 提前熟悉小游戏审核规范，避免敏感内容 |
| 外挂/作弊 | 中 | 关键逻辑放服务器，客户端只展示；加密通信 |
